## 美股简报（16:00）详细结构

```
1. 市场总览
   - VIX当前值 + 变化幅度
   - 四大指数：道指/标普500/纳指/罗素2000 当日涨跌幅

2. 板块强弱（11个SPDR行业ETF代理）
   - 🔥 最强3个板块
   - 🧊 最弱3个板块

3. 主题动态（每次独立AI联网搜索，不与21:30报告做内容对比）
   - 🤖 AI/半导体重要新闻
   - 📑 今日重要财报
   - 🌍 中东局势/中美关系最新进展

4. 个股清单（15只美股逐一展示）
   - 实时价格、涨跌幅
   - 技术面简析（RSI/MA20/MA50/量比，规则计算，非完整MFTSR）
   - 新闻摘要（AI联网搜索）+ 简短操作倾向
```

> 📌 说明：板块强弱用11个SPDR行业ETF（XLK科技/XLE能源/XLF金融等）的当日涨跌幅排名作为代理指标，因为yfinance没有现成的板块指数。

> 📌 说明：三大主题动态**每次都是独立搜索总结**，不会跟前一晚21:30的深度报告做逐条对比（按你的要求简化处理）。如果之后想要真正的"增量对比"（即只播报新发生的变化），需要加存储机制，可以后续升级。

## 文件结构

```
market-agent/
├── shared_config.py              ← 股票池/ETF/指数/板块清单的唯一来源
├── a_share_report.py             ← A股早报
├── us_midday_brief.py            ← 美股简报（已扩充）

    └── workflows/
        ├── a_share_report.yml

        └── us_close_report.yml
```

## 第四步：测试（⚠️ 这一步不能跳过）

上传文件之后，**必须**做以下两件事，否则可能出现"文件传了但实际运行时间不对"的情况
（之前发生过：上传后没检查，结果三个报告在完全错误的时间触发，UK 9:41/14:41/22:21，
跟任何配置都不符——最可能是仓库里残留旧版workflow文件，新文件没有真正生效）：

1. **检查Actions页面显示的schedule是否正确**：进入仓库 → Actions → 点开某个workflow
   名字（如"A股早报"）→ 右上角点击 **...** → 应该能看到"This workflow has 1 schedule
   trigger"或类似信息，确认cron表达式与你预期的一致。
2. **手动触发一次确认能跑通**：点击 **Run workflow**，等运行完成（绿勾✅），去Telegram
   核实确实收到了消息，时间戳也是合理的。

只有以上两步都确认无误，才能放心等待下一次自动触发；否则一旦时间不对，可能会持续
按错误时间运行很多天都不会被发现。

Actions标签 → 依次手动触发3个workflow测试。美股简报因为涉及更多搜索调用（3个主题+15只股票=18次搜索调用），运行时间会比之前长，预留 `timeout-minutes: 20`。
