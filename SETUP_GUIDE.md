## 美股简报（16:00）详细结构

```
1. 市场总览
   - VIX当前值 + 变化幅度
   - 四大指数：道指/标普500/纳指/罗素2000 当日涨跌幅

2. 板块强弱（11个SPDR行业ETF代理）
   - 🔥 最强3个板块
   - 🧊 最弱3个板块

3. 主题动态（每次独立AI联网搜索，不与21:30报告做内容对比）
   - 🤖 AI/半导体重要新闻
   - 📑 今日重要财报
   - 🌍 中东局势/中美关系最新进展

4. 个股清单（15只美股逐一展示）
   - 实时价格、涨跌幅
   - 技术面简析（RSI/MA20/MA50/量比，规则计算，非完整MFTSR）
   - 新闻摘要（AI联网搜索）+ 简短操作倾向
```

> 📌 说明：板块强弱用11个SPDR行业ETF（XLK科技/XLE能源/XLF金融等）的当日涨跌幅排名作为代理指标，因为yfinance没有现成的板块指数。

> 📌 说明：三大主题动态**每次都是独立搜索总结**，不会跟前一晚21:30的深度报告做逐条对比（按你的要求简化处理）。如果之后想要真正的"增量对比"（即只播报新发生的变化），需要加存储机制，可以后续升级。

## 文件结构

```
market-agent/
├── shared_config.py              ← 股票池/ETF/指数/板块清单的唯一来源
├── a_share_report.py             ← A股早报
├── us_midday_brief.py            ← 美股简报（已扩充）
├── us_close_report.py            ← 美股收盘完整报告
└── .github/
    └── workflows/
        ├── a_share_report.yml
        ├── us_midday_brief.yml
        └── us_close_report.yml
```

**重要**：4个 `.py` 文件必须都在仓库根目录，因为互相用 `from shared_config import ...` 引用。

---

## 第二步：上传文件

1. 创建/使用私有仓库 `market-agent`
2. 上传4个Python文件到**仓库根目录**
3. 上传3个workflow文件到 `.github/workflows/`

> 💡 如果你是从之前的v3版本（含事件监控）升级：删除仓库里旧的 `event_monitor.py` 和 `.github/workflows/event_monitor.yml`，否则旧workflow仍会按原计划运行。

---

## 第三步：配置Secrets

仓库 → Settings → Secrets and variables → Actions → New repository secret，添加4个密钥（同上表）。

---

## 第四步：测试

Actions标签 → 依次手动触发3个workflow测试。美股简报因为涉及更多搜索调用（3个主题+15只股票=18次搜索调用），运行时间会比之前长，预留 `timeout-minutes: 20`。

## 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 美股简报运行时间变长 | 新增板块/指数/主题搜索 | 正常，预留了20分钟超时 |
| 板块强弱显示"数据不可用" | 个别SPDR ETF代码yfinance偶尔限流 | 脚本会跳过该板块，不影响其余排名 |
| 主题动态总是"暂无新进展" | 该主题确实没有重大新闻，或搜索未命中 | 正常现象，AI被要求不编造内容 |
| A股数据缺失 | yfinance对A股支持有限 | 已知限制，标注"数据不可用"而非虚假数字 |
